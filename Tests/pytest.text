# 🧠 1. What is a Unit Test (in ML systems)

A **unit test** checks a *single small piece of logic* in isolation.

### In your recommendation system:

| Component       | Unit Test Target             |
| --------------- | ---------------------------- |
| feature builder | input → feature vector shape |
| ranker          | sorting logic                |
| retrieval       | FAISS output validity        |
| business rules  | filtering logic              |
| cache           | store/retrieve correctness   |

👉 Key idea: **No full pipeline, no API calls, no DB dependency**

# 🧪 3. Example Unit Tests (Your System)

---

## ✅ A. Ranker Unit Test

### File: `tests/test_ranker.py`

```python
import numpy as np
from app.ranking.ranker import Ranker


def test_ranker_output_length():
    ranker = Ranker()

    X = np.random.rand(10, 20)
    candidates = list(range(10))

    output = ranker.rank(X, candidates)

    assert len(output) == 10
```

---

### 🔍 What this tests:

* ranker runs without crashing
* output size matches input size
* no missing items

---

## ✅ B. Retrieval Unit Test

```python
import numpy as np
from app.retrieval.candidate_generator import CandidateGenerator


def test_retrieval_returns_results():
    cg = CandidateGenerator()

    embedding = np.random.rand(64)

    results = cg.generate(embedding)

    assert len(results) > 0
```

---

### 🔍 What this tests:

* FAISS index is loaded
* ANN search works
* system returns candidates

---

## ✅ C. Business Rules Test

```python
from app.reranking.business_rules import BusinessRulesEngine


def test_business_rules_filtering():
    engine = BusinessRulesEngine()

    items = [
        {"id": 1, "score": 0.9},
        {"id": 2, "score": -1},  # invalid
        {"id": 3, "is_expired": True}
    ]

    filtered = engine.apply(items)

    ids = [x["id"] for x in filtered]

    assert 2 not in ids
    assert 3 not in ids
```

---

### 🔍 What this tests:

* filtering logic
* business constraints
* safety rules

# 🚀 4. How to RUN Unit Tests

---

## ▶️ Run ALL tests

From project root:

```bash
pytest
```

---

## ▶️ Run specific file

```bash
pytest tests/test_ranker.py
```

---

## ▶️ Run specific test

```bash
pytest tests/test_ranker.py::test_ranker_output_length
```

---

## ▶️ Verbose output (recommended)

```bash
pytest -v
```

---

## ▶️ Show prints (debug mode)

```bash
pytest -s
```

---

# 🧠 5. What Happens When You Run pytest

Pytest automatically:

```text
1. finds files named test_*.py
2. finds functions named test_*
3. executes them
4. reports pass/fail
```

Example output:

```text
tests/test_ranker.py::test_ranker_output_length PASSED
tests/test_retrieval.py::test_retrieval_returns_results PASSED
```

# 🧠 6. Senior-Level Best Practices

This is what separates juniors from seniors.

---

## ✅ 1. Tests must be deterministic

❌ bad:

```python
np.random.rand()
```

✔ better:

```python
np.random.seed(42)
```

---

## ✅ 2. Avoid real DB / network in unit tests

Use mocks:

```python
from unittest.mock import MagicMock
```

---

## ✅ 3. Keep tests fast (<1 sec each)

Unit tests should be **instant**

---

## ✅ 4. One test = one behavior

❌ bad:

* test everything together

✔ good:

* one function → one test

---

## ✅ 5. Assert behavior, not implementation

❌ bad:

```python
assert model.weights is not None
```

✔ good:

```python
assert output.shape[0] == len(input)
```

---

# 🧠 7. Unit Test vs Other Tests (Very Important)

| Type             | Scope           | Speed   |
| ---------------- | --------------- | ------- |
| Unit test        | single function | fast ⚡  |
| Integration test | full pipeline   | medium  |
| E2E test         | full system     | slow    |
| Load test        | performance     | slowest |

They ensure:

* system stability
* no silent crashes
* correct data flow
* safe deployments
