Designing **persistent memory for agents** is a systems problem: you’re building something closer to a **personal knowledge system + retrieval engine + write policy**, not just a database.

A senior framing is:

> Persistent memory = (what to store) + (how to store it) + (how to retrieve it) + (how to keep it clean over time)

---

# 1. What is persistent memory?

Persistent memory is an external system that allows an agent to:

* remember across sessions
* personalize behavior
* accumulate knowledge over time
* retrieve relevant past experiences

Unlike short-term memory (context window), it is:

* durable
* structured
* queryable

# 3. Types of persistent memory (you must design all 3)

## 3.1 Semantic memory (facts & knowledge)

Examples:

* “User works on RAG systems”
* “Project uses FAISS”

Stored as embeddings + metadata.

---

## 3.2 Episodic memory (events/history)

Examples:

* “User asked about tool hallucination on May 16”
* “User debugged latency issue”

Stored as logs or vectors.

---

## 3.3 Procedural memory (behavior/preferences)

Examples:

* “Always respond concisely”
* “Prefer Python code examples”

Stored as structured key-value.

---

# 4. Memory write pipeline (CRITICAL)

Not everything should be stored.

## Step 1: Memory candidate generation

LLM extracts:

```json id="m1"
{
  "type": "preference",
  "content": "User prefers concise explanations",
  "importance": 0.9
}
```

---

## Step 2: Memory filter (gatekeeper)

Decides:

* store / reject / summarize

Rules:

* store only stable or reusable info
* ignore one-off queries

---

## Step 3: Memory enrichment

Add metadata:

* timestamp
* source
* confidence
* importance score

---

## Step 4: Storage

Stored in:

* vector DB (semantic)
* KV store (facts)
* log store (history)

---

# 8. Memory consolidation (VERY IMPORTANT for scale)

Without this, memory becomes noisy.

## Techniques:

### 8.1 Summarization

Convert many logs into one fact:

---

### 8.2 Deduplication

Merge similar memories.

---

### 8.3 Clustering

Group related memories before storage.

# 13. Common mistakes

❌ storing everything (memory pollution)
❌ no retrieval ranking
❌ no decay mechanism
❌ no write filter
❌ injecting irrelevant memories
❌ mixing episodic + semantic memory

---

# 14. Senior-level interview answer

Persistent memory in agentic systems is designed as a multi-layer architecture consisting of semantic memory (vector databases), structured key-value memory, and episodic event logs. A memory write filter determines what information is worth storing based on importance, stability, and reuse potential.

At inference time, a memory retrieval engine performs similarity search combined with metadata filtering and ranking based on relevance, recency, and importance before injecting relevant memories into the model context. Additional mechanisms such as memory consolidation, deduplication, and decay functions are used to ensure scalability and prevent noise accumulation over time.
